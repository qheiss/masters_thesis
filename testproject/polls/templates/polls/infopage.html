How to run the algorithm: Detailed description

With the Clustering web application, you can run a bi-clustering algorithm either on pre-defined datasets (Lung or Breast Cancer) or on your own uploaded datasets. Registered users can also store their own uploaded data and re-use them. Your uploaded data must contain one file with expression data and one with PPI data.

These expression data must have the following format:

The second column, labeled as 'prognosis' above, can also be labeled 'cancer_type' and contain information about the cancer subtype. Each row consists of gene expression values of a list of genes for a specific patient.

Where each row consists of two NCBI gene IDs of the interacting genes.


Algorithm parameters
In the web application, you can specify the minimum and maximum number of genes you want per cluster. Also, you can choose how many iterations the algorithm should perform at most; for the settings used in the example runs, the algorithm usually converged between run 25 and 45.
If you wish to set more parameters, you can select the number of ants, evaporation rate, pheromone significance, HI significance and the stop criteria.
While the algorithm is running, a figure will be shown displaying the algorithm's progress (convergence behaviour).


Results
After running the analysis, the results will be visualized. A heatmap displays patient and gene clusters and gene expression levels; in the PPI graph, you can see all interactions between genes in the result with the node for each gene colorized according to the expression level difference between both patient clusters. Additionally, at the bottom of the site you will see the convergence behaviour of the algorithm for your run.
